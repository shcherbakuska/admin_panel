1.1 Архитектура  
Сервис состоит из следующих служб:  
**Frontend (React)** - отвечает за визуализацию данных, получает данные через REST API и реализует пагинацию по страницам. Основные компоненты:  
−	App.jsx – главный файл, соединяющий в себе связь с бэкендом приложения, а также отображение таблицы студентов и пагинации на странице;  
−	Students.jsx – файл отображения базы данных в виде таблицы;  
−	Pagination.jsx – файл, отвечающий за верное отображение пагинации на странице.  
**Backend (FastAPI) **- предоставляет REST API для работы с базой данных, взаимодействует с базой данных через ORM (SQLAlchemy для PostgreSQL).  
**Database (PostgreSQL)** – хранит базовую информацию о студентах для дальнейшего отображения на панель администратора.  
**Nginx **- используется для раздачи статических файлов фронтенда, а также связывания бэкенда и фронтенда (во избежания ошибки CORS)  
**Docker** – позволяет завернуть все микросервисы (frontend, backend и база данных) зв контейнеры. Docker Compose обеспечивает лёгкий запуск проекта.  

1.2	Структура базы данных  
База данных Postgresql состоит из одной таблицы students, которая ии будет отображаться на нашем сайте. Структура таблицы students:    
−	id – уникальный номер пользователя, первичный ключ;  
−	name – имя;  
−	surname – фамилия;  
−	partonymic – отчество;  
−	course – курс;  
−	group – группа;  
−	faculty – факультет.  

1.3	Описание протоколов  
REST API использует JSON для обмена данными между сервером и клиентом, при этом взаимодействие осуществляется через стандартный HTTP-протокол.  
1.4	Описание API  
REST API:  
−	GET /api/students?skip&limit – получить студентов с определенным началом и в определенном количестве;  
−	POST /api/student – добавить запись о студенте;  
−	DELETE /api/students/{student_id} – удалить запись о студенте.
